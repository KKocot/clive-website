## Why SVG-only OG (for now)

Twitter, Facebook, LinkedIn and Slack all accept `image/svg+xml` for `og:image`,
but support is not 100% — older Slack and some link previewers downgrade to a
gray box. Until we generate a PNG, the SVG is set as the canonical OG image and
declared with explicit `og:image:width=1200` / `og:image:height=630` so rich
previews still get correct aspect ratios.

## Generating `og-default.png`

Pick one of:

### Option A — Local screenshot (manual, zero new deps)

1. Open `docs/og-template.html` in a Chromium-based browser.
2. DevTools  Toggle device toolbar  set viewport to 1200x630, DPR 1.0.
3. Right-click the `.og` element  "Capture node screenshot".
4. Save as `public/og-default.png`. Optimize with `pngcrush` or `oxipng`.

### Option B — Playwright (repeatable)

```bash
# requires Playwright already in the project; do NOT add a new dep just for this
node -e "
const { chromium } = require('playwright');
(async () => {
  const browser = await chromium.launch();
  const page = await browser.newPage({ viewport: { width: 1200, height: 630 } });
  await page.goto('file://' + process.cwd() + '/docs/og-template.html');
  const el = await page.\$('.og');
  await el.screenshot({ path: 'public/og-default.png', omitBackground: false });
  await browser.close();
})();
"
```

### Option C — Vercel OG (build-time, future)

Migrate to `@vercel/og` (Satori) once we have per-post OG generation. Out of
scope for v1 — needs a `decisions` entry in the Knowledge Base before adding.

## Generating `apple-touch-icon.png`

Pre-iOS 14 Safari does not honor SVG icons reliably. Generate a 180x180 PNG
from `favicon.svg`:

```bash
# preferred — uses sharp which is already in the project
node -e "
require('sharp')('public/favicon.svg')
  .resize(180, 180)
  .png({ compressionLevel: 9 })
  .toFile('public/apple-touch-icon.png');
"
```

Then verify by visiting the site on an iPhone and "Add to Home Screen".

- [ ] `dist/robots.txt` exists and points to `sitemap-index.xml`
